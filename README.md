# Sales-Performance-Analysis

## Objective
The objective of this project is to analyze e-commerce sales data to identify revenue trends, top-performing products, and high-value customers, enabling better business decision-making.

Tools Used
- MySQL
- SQL (CTEs, Window Functions, Joins)

## Dataset
Source: Kaggle â€“ E-Commerce / Superstore Sales Dataset  
The dataset contains order-level sales, customer, product, and category information.

## Key Analysis Performed
- Monthly and yearly revenue trends
- Top products and categories by profit
- Customer purchase behavior analysis
- Ranking customers based on total spending
- Running total of sales using window functions

## SQL Concepts Used
- INNER & LEFT JOINS
- GROUP BY and HAVING
- Common Table Expressions (CTEs)
- Window Functions (RANK, SUM OVER)
- Date functions

## Key Insights
- A small percentage of customers contribute to a majority of revenue
- Certain categories generate high sales but low profit margins
- Sales show strong seasonality during year-end months

## Business Recommendations
- Focus marketing efforts on high-value repeat customers
- Optimize pricing strategies for low-margin products
- Improve inventory planning for peak seasons
